Использование API ER:LC России связано с рядом обязательств. Независимо от того, создаёте ли вы Discord-бота, сайт бизнеса или любую другую интеграцию, вы должны строго соблюдать эти правила, чтобы сохранить доступ к API и защитить наше сообщество.

## Область Поддержки

**Мы предоставляем техническую поддержку по API исключительно разработчикам, имеющим прямой доступ к своему коду**. Если вы создаёте продукт для конечных пользователей (например, бот для других серверов или своего бизнеса), вы несёте ответственность за их поддержку самостоятельно. Команда разработки ER:LC России не будет решать проблемы пользователей, которые просто используют ваш сервис.

## Запрещённые Действия

<Danger>
  Следующие действия **строго запрещены** и приведут к **бессрочному отзыву** доступа к API (блокировке токенов), а также к возможным банам на наших игровых серверах и в Discord:

  * **Нюкинг, рейды или краш** серверов и Discord-каналов вашим приложением.
  * **Абуз командами и багами** нашего API.
  * **Выдача себя за** других игроков, администрацию или разработчиков проекта.
  * **Распространение дезинформации**, ложных сведений или слив конфиденциальных данных.
  * **Угрозы и шантаж** пользователей в любой форме.
  * **Спам, флуд и DDoS-атаки** на эндпоинты API или связанные сервисы.
  * Любые другие деструктивные действия, совершённые с вредоносными намерениями.
  * **Несоблюдение [официальных правил API ER:LC](https://apidocs.erlc.gg/policies/aups)** при интеграции нашего API для ER:LC.
</Danger>

### Правила Сообщества ER:LC Россия

API не может использоваться для действий, нарушающих Правила Сообщества ER:LC. Они доступны [**по этой ссылке**](https://wiki.erlcrussia.com).

## Безопасность Данных и Ключей Доступа

Вы несёте полную ответственность за безопасность ваших API-ключей, токенов авторизации и любых данных, которые возвращает API.

* Не публикуйте ключи в открытых репозиториях (GitHub, GitLab и др.).
* Не оставляйте их в клиентском коде (фронтенде), где их может скопировать любой пользователь.
* Если ваш ключ был скомпрометирован или утёк, **немедленно сообщите нам об этом через тикет в Discord**. Игнорирование утечки или несвоевременное уведомление администрации может привести к **бессрочной блокировке** вашего доступа к API.

## Оптимизация и Нагрузка

Если ваш сервис обрабатывает большой объём запросов, вы обязаны реализовать адекватные меры **кэширования данных**, **ограничения частоты запросов** и **корректной обработки ошибок (например, 429 Too Many Requests)**. 

Мы оставляем за собой право временно или навсегда ограничить доступ для любого приложения, которое перегружает сервер, блокирует Event Loop, снижает производительность инфраструктуры или создаёт угрозу стабильности проекта.

Если вы обнаружили критическую ошибку, баг на обход ограничений или уязвимость безопасности, затрагивающую API ER:LC России или его серверную часть, немедленно сообщите об этом через тикет в Discord или на почту [**pancakeboyman_8@erlcrussia.com**](mailto:pancakeboyman_8@erlcrussia.com).
