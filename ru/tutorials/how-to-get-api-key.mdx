title: "Как получить API ключ?"

---

Для интеграции с нашими сервисами вам понадобятся соответствующие доступы и ключи авторизации. Обратите внимание, что доступ к HTTP API, WebSocket-серверам и управлению вебхуками разделён из соображений безопасности.

## Получение Доступа

<Steps>
  <Step title="Открытие Тикета">
    Для первичного получения доступа к HTTP API, WebSocket или к системе управления вебхуками вам необходимо **открыть тикет на нашем Discord-сервере**. Опишите ваш проект и укажите, какие именно возможности и доступы вам понадобятся для реализации.
  </Step>
  <Step title="Одобрение Заявки">
    Дождитесь, пока администрация проекта рассмотрит ваше обращение. После одобрения вашей заявки к вашему Discord-аккаунту будут привязаны соответствующие права доступа.
  </Step>
  <Step title="Просмотр и Управление Ключами">
    Как только права будут выданы, вы сможете генерировать, просматривать и управлять своими токенами прямо внутри Discord-сервера с помощью слэш-команд.
  </Step>
</Steps>

## Доступные Типы Ключей и Команды

### 1. HTTP API Ключ
Основной токен, который используется для отправки стандартных HTTP-запросов к нашим эндпоинтам. Передаётся в заголовке `Authorization`.
* **Команда для просмотра:** `/апи просмотр`

### 2. WebSocket Ключ
Для работы с событиями в реальном времени используется **отдельный независимый ключ**. Не пытайтесь использовать стандартный HTTP-токен для подключения к сокетам — сервер его отклонит.
* **Команда для просмотра:** `/вебсокет просмотр`
<Danger>
  **Всегда держите ваши ключи в секрете. Ключи от WebSocket и HTTP API абсолютно автономны и управляются независимо друг от друга.**
</Danger>

## Управление Вебхуками

Если вашему сервису необходимо получать автоматические уведомления о событиях на сервере, вы можете настроить вебхуки. 

Доступ к панели управления вебхуками также выдаётся **строго через тикеты** у администрации проекта. Как только права будут привязаны к вашему аккаунту, вам станет доступна специальная команда для настройки:

* **Команда для управления:** `/вебхук управление`

<Tip>
  **При использовании этой команды бот откроет удобное меню, где вы сможете привязывать, удалять и просматривать активные URL-адреса для отправки вебхуков.**
</Tip>
