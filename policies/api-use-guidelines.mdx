**ER:LC Россия не оказывает поддержку пользователям сторонних проектов, работающих на этом API. Помощь предоставляется только тем, кто имеет доступ к исходному коду.**

### Основные Правила Использования

- Использовать API в злонамеренных целях строго запрещено. К таким действиям относятся (но не ограничиваются): сносы, рейды, распространение ложной информации, угрозы, спам и другие вредоносные действия. Нарушение этих правил приведёт к пожизненному отзыву доступа к API и, при необходимости, блокировке на наших серверах (ER:LC, Discord, Minecraft и т.п.).
- Все **ключи доступа к API**, **ключи серверов** и любые **данные API** должны храниться в безопасности. Не делитесь ими с третьими лицами.
- Запрещено использовать не задокументированные эндпоинты — такие действия отслеживаются и могут повлечь наказание.
- Приложения с **крупной нагрузкой** должны реализовать:
  - Ограничение частоты запросов (Rate Limiting).
  - Корректную обработку ошибок.
  - Механизмы защиты от злоупотреблений.
  - Меры безопасности для защиты данных пользователей.
- Мы оставляем за собой право **ограничить или приостановить** доступ, если приложение:
  - Ухудшает производительность API.
  - Ставит под угрозу безопасность.
  - Создаёт чрезмерные риски для нашего сообщества.
- Использование API **не должно нарушать ЛЮБЫЕ наши правила**.

# Правила использования API

> Правила и ответственность при доступе к API приватного сервера ER:LC, включая запрещённые действия, безопасность ключей и стандарты сообщества.

Использование API приватного сервера ER:LC связано с рядом обязательств. Независимо от того, создаёте ли вы Discord-бота, веб-панель или любую другую интеграцию, вы должны соблюдать эти правила, чтобы сохранить доступ к API и защитить сообщество ER:LC.

## Область поддержки

**Мы предоставляем поддержку API исключительно разработчикам, имеющим прямой доступ к своему коду**. Если вы создаёте продукт для конечных пользователей, вы несёте ответственность за их поддержку самостоятельно. ER:LC не будет решать проблемы от имени людей, которые просто используют ваше приложение.

## Запрещённые действия

<Danger>
  Следующие действия **строго запрещены** и приведут к **бессрочному отзыву** доступа к API, а также к возможным банам в игре ER:LC:

  * **Нюкинг или рейды** приватных серверов
  * **Выдача себя за** игроков, сотрудников или персонал PRC
  * **Распространение дезинформации или ложных сведений**
  * **Угрозы пользователям** в любой форме
  * **Спам** на серверах, игроках или самом API
  * Любые другие действия с API, совершённые недобросовестно или с вредными намерениями
</Danger>

## Защита учётных данных и данных

Вы несёте ответственность за безопасность ключей доступа к API, ключей сервера и любых данных, возвращаемых API. Не раскрывайте ключи в публичных репозиториях, клиентском коде или в любом месте, доступном неавторизованным лицам. Если ключ был скомпрометирован, немедленно выполните его ротацию.

## Требования для крупномасштабных приложений

Если ваше приложение выполняет большой объём запросов, вы должны реализовать адекватные меры **ограничения частоты запросов**, **обработки ошибок**, **предотвращения злоупотреблений** и **защиты данных**. PRC оставляет за собой право приостановить или ограничить доступ для любого приложения, которое снижает производительность сервиса, ставит под угрозу безопасность или создаёт необоснованный риск для сообщества.

Ознакомьтесь с разделом [Глобальные ключи API для крупномасштабных приложений](/guidelines/large-scale-apps) для получения инструкций по обработке больших объёмов запросов и подаче заявки на глобальный ключ API.

## Прекращение автоматизации после удаления

Когда ваш сервис удаляется с приватного сервера, вы должны немедленно прекратить выполнение API-запросов с использованием ключа этого сервера.

Пример: Если вашего бота удалили из Discord-сервера, вы должны прекратить все API-запросы, использующие ключ сервера, полученный от этого сервера.

### Правила сообщества PRC

API не может использоваться для действий, нарушающих Правила сообщества ER:LC. Они доступны в канале `#community-guidelines` на Discord-сервере PRC.

## Сообщение о проблемах безопасности

Если вы обнаружили уязвимость безопасности, затрагивающую API ER:LC или его инфраструктуру, немедленно напишите на [**flatbird@erlc.gg**](mailto:flatbird@policeroleplay.community). Не раскрывайте уязвимость публично до обращения к нам.
